# SheetSync Learnings

## Technology Stack
- Next.js 16 (Canary) with Turbopack.
- React 19.
- NextAuth for Google-only authentication.
- TanStack Table v8 + TanStack Virtual for large dataset handling.

## Key Patterns
- **Next.js 16 Dynamic Routes:** `params` in Page and Route handlers is now a Promise and must be awaited or used with the `use()` hook.
- **Dependency Management:** React 19 requires `--legacy-peer-deps` for many current ecosystem libraries (e.g., `lucide-react`, `next-auth`) until they officially release v19-compatible versions.
- **Supabase Integration:** Using a custom server client to sync NextAuth sessions with a `profiles` table for RLS consistency.

## Build Process
- Always use `npm run build` locally to catch TypeScript and Next.js 16 specific errors (like the async `params` change).
